# Contributing

Guidance for changing this repo rather than consuming it. It splits into two
concerns: **development** (working on the code and its conventions) and
**documentation** (authoring this docs site). Agents working inside the docs app
should also read `documentation/AGENTS.md` for the runtime authoring contract.

## Contents

- [Development](development/index.md) — Dev workflow, verification commands, repository conventions, commit conventions, and git hooks/safety.
- [Documentation](documentation/index.md) — The docs authoring contract: navigation rules, supported Markdown features, and the docs-change review checklist.
