### Important

- **Fan-in rejects valid default output paths outside the fan-in cwd** (`plugins/consensus/skills/consensus-refine/scripts/consensus-refine.mjs:1267`)
  - Issue: `fanInParallelRun` sets the trusted root to `cwd` unless `--allow-root` is supplied, and `normalizeParallelManifest` then validates `manifest.output_path` only against that root. A valid `--prepare-parallel` run can still produce the default output as `<input>.consensus.md` next to an absolute input file outside `cwd`; fan-in then rejects its own prepared manifest with `PATH_OUTSIDE_ROOT`. The design's path-safety table allows default output next to the input, while explicit `--output` is the domain that must stay under `cwd` or `--allow-root`.
  - Fix: Preserve the prepared output domain in the manifest, or validate fan-in output paths against the same rules used by prepare: default output is allowed only when it equals `path.resolve(`${manifest.input_path}.consensus.md`)` and resolves under the input's real parent; explicit output remains confined to `cwd` or `--allow-root`. Add a regression test that prepares with an absolute input outside `cwd`, omits `--output`, writes simulated section outputs, and successfully fans in without requiring `--allow-root`.
