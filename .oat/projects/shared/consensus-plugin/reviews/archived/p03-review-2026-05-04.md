### Important

- **Fan-in trusts manifest-controlled file paths outside confinement** (`plugins/consensus/skills/consensus-refine/scripts/consensus-refine.mjs:1208`)
  - Issue: `fanInParallelRun` reads `entry.output_section`, `entry.output_records`, `entry.output_status`, and fallback `entry.section_file` directly from the manifest, then writes `manifest.output_path` with `path.dirname(manifest.output_path)` as the default allowed root. A tampered or foreign manifest can therefore choose arbitrary read paths and its own write root, instead of being confined to the prepared run directory and the explicit output/allow-root contract. This conflicts with the design's path-safety rules for run files and explicit outputs.
  - Fix: Validate the manifest schema/type up front, resolve `manifest.run_dir` and all section file paths against a trusted root (`cwd` unless `--allow-root` is provided), require section inputs/records/status/output files to remain under the prepared run directory, and confine `manifest.output_path` to `cwd` or `--allow-root` rather than to its own dirname. Add a regression test with a tampered manifest path escaping the run root.
  - Requirement: FR4 / design path safety
