# Project Summary: docs-ia

## Overview

`docs-ia` (backlog **bl-ecaa**) stood up a dedicated documentation site for the
skills repo and slimmed the increasingly dense `README.md` to a lean entry point.
The README had grown to ~205 lines as the consensus plugin, provider CLI, and
skill surface expanded; this project moved that reference content into a
navigable, audience-organized site so the README can stay an entry point and
future projects document themselves into the site (via `oat-project-document`)
rather than the README.

Run as a quick-mode project with a collaborative lightweight design pass for the
information architecture. Framework decision: **Fumadocs** (DR-025) — chosen for
toolchain consistency with the repo's Node/pnpm/TypeScript stack over the
Python-based MkDocs path.

- **Two-trunk audience IA** (24 pages, build green):

    evaluate, configuration), `skills/` (session-observer, export-session-transcript).

    `repository-layout`, `contributing/` split into `development/` + `documentation/`
